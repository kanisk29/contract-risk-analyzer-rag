# Contract Risk Analyzer (RAG)

An automated system for analyzing legal contracts and identifying high-risk clauses under the framework of Indian Contract Law. The system uses a Retrieval-Augmented Generation pipeline with vector search, cross encoder reranking and LLM reasoning to produce structured risk assessments.

## Disclaimer

This project is developed for **educational and demonstration purposes only**.

- The analysis generated by this system is **not legal advice**  
- Results may be incomplete, inaccurate, or not applicable to specific cases  
- The system does not replace consultation with a qualified legal professional  
- Users should **not rely on this tool for legal decisions**

Always consult a licensed legal expert before acting on any contract-related matters.

## Features

- Clause-level risk detection across contract text  
- Risk classification into LOW, MEDIUM, and HIGH  
- Legal grounding using principles from the Indian Contract Act, 1872    
- Vector similarity search with MMR-based diversification and cross-encoder reranking

## Tech Stack

| Component          | Technology        |
|-------------------|------------------|
| Framework         | FastAPI          |
| Orchestration     | LangChain        |
| Vector Database   | ChromaDB         |
| LLM               | Groq (LLaMA 3)   |
| Reranking         | Cohere Rerank    |
| Embeddings        | HuggingFace      |

## System Architecture


```
User Input (PDF/Text)
        ↓
Clause Segmentation
        ↓
Vector Retrieval 
        ↓
LLM Analysis (Groq - LLaMA 3)
        ↓
Risk Classification + Legal Reasoning
        ↓
Contract Score + Summary
```

## Deployment

The system is deployed using **Render** with separate services:

- **Backend (FastAPI)** → API for contract analysis  
- **Frontend (Streamlit)** → User interface  

## Analysis Example:

### Input Clause:

The employee shall not work with any competitor for 3 years after termination.

## Key Design Decisions

- Hybrid retrieval improves accuracy over pure vector search  
- Parallel processing reduces latency for multi-clause contracts  
- External LLM (Groq) avoids local GPU dependency  
- CPU-based embeddings for lightweight deployment  

---

## Limitations

- Cold start latency on free hosting tiers  
- Retrieval quality depends on knowledge base coverage  
- Legal reasoning is assistive, not a substitute for professional advice  

---
